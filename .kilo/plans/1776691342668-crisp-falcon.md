# MTN (Mantori Trading Navigator) 프로그램 리뷰

## 1. 프로그램 개요

**프로젝트명:** Mantori Trading Navigator (MTN)  
**목적:** 개인 투자자를 위한 추세추종/성장주 투자 전략 지원 프로그램  
**기술 스택:** Next.js 16, TypeScript, Supabase, TailwindCSS

## 2. 핵심 기능 분석

### 2.1 CAN SLIM 엔진 (`lib/finance/engines/canslim-engine.ts`)
 William O'Neil의 CANSLIM 7가지 기준을 구현:
- **M (Market):** 시장 방향성 - 분배일/FTD 기반 매크로 판단
- **C (Current):** 분기 실적 - EPS 성장률, 매출성장률, 3분기 연속 성장
- **A (Annual):** 연간 실적 - ROE, 연도별 EPS 성장, 적자 이력 검증
- **N (New):** 신고가/베이스 패턴 - 52주 고가 근접, 피벗 돌파
- **S (Supply):** 수급 - 거래량, Float大小
- **L (Leadership):** 상대강도 - RS Rating 80+
- **I (Institutional):** 기관 수급 - 보유 추세, 기관 수

**장점:**
- 7가지 기준의 순차적 검증으로 빠른 탈락 처리
- 분배일 기반 시장 상태 자동 반영
- Confidence 레벨 (HIGH/MEDIUM/LOW) 제공

**주의사항:**
- 펀더멘털 데이터 의존도가 높아 데이터 수집 지연 가능성
- 한국 markets에 적용시 실적 데이터 수집 한계

### 2.2 VCP 엔진 (`lib/finance/engines/vcp-engine.ts`)
 Mark Minervini의 Volatility Contraction Pattern:
- **Contraction Detection:** 2~6개 점진적 수축 단계 감지
- **Volume Dry-Up:** 거래량 감소 추세 분석
- **BB Squeeze:** 볼린저 밴드 수축 상태 감지
- **Pocket Pivot:** 기관 매집 초기 시그널
- **High Tight Flag:** 단기간 강한 모멘텀 패턴 분석

**장점:**
- 4개 레이어의 종합 점수 (0-100)
- 등급 분류 (strong/forming/weak/none)
- 피벗/무효화 가격 자동 계산

### 2.3 스캐너 (`app/scanner/page.tsx`)
- Multi-universe 지원: NASDAQ100, SP500, KOSPI100, KOSDAQ100
- SEPA/VCP 신호 동시 판단
- Real-time 진행 상황 표시
- Macro trend 기반 필터링 (FULL/REDUCED/HALT)

### 2.4 포트폴리오 리스크 관리 (`lib/finance/core/portfolio-risk.ts`)
- 자본금 기반 최대 포지션 수 제한 (100만=2개, 1000만=5개, 1000만+=10개)
- 총.open risk 8% 초과 경고
- 섹터 집중도 35% 이상 경고

### 2.5 거래 메트릭스 (`lib/finance/core/trade-metrics.ts`)
- R-Multiple 계산 (손익/Planned Risk)
- Entry Slippage 추적
- realized/unrealized P&L 동시 추적
- Trailing stop 지원

## 3. 데이터 흐름

```
Universe API → KIS 가격 조회 → Yahoo Fallback → Benchmark 조회 → SEPA/VCP 계산 → 추천 결과
```

**데이터 sources:**
- 한국: KIS (Korea Investment Securities) API
- 미국: Yahoo Finance API
- 펀더멘털: DART (금융감독원 전자공시), SEC EDGAR

## 4. 주요 UI 페이지

## 5. 설계的长点

1. **이중 스크리너:** CAN SLIM + VCP cross-validation으로 티어 분류
2. **Automation:** Batch 스캔 및 RS Metrics 계산 자동화
3. **AI 통합:** Gemini 기반 market insight 생성
4. **Telegram 연동:** 알림 및 webhook 지원
5. **Database:** Supabase 기반 완전한 데이터 영구 저장

## 6. 개선 가능 영역

### 6.1 추세추종 특화 기능 부족
현재 CANSLIM/VCP 중심이므로 전통적인 추세추종 (이동평균 교차, MACD 등) 관련 기능이 거의 없음

**개선 방안:**
- 이동평균 (50일, 200일) 기반 추세 필터 추가
- Trend strength indicators (ADX, Aroon) 도입
- Trailing stop automation (MA-Based, ATR-Based)

### 6.2 리스크 관리
- 손절 규칙이 수동 중심 (7~8% 기계적 손절만 기본)
- Partial profit taking ($2R, $3R 등) automation 미흡
- 포지션 스케일링 ($in/out) 미지원

### 6.3 백테스트 부재
历史 데이터 기반 검중 기능 없음

### 6.4 한국 markets 특화
- KIS_API 연결 필요 (개인 정보)
- 증거금 거래 미검토

## 7. 결론 및 권장

**적합 대상:**
- William O'Neil CANSLIM 방법론 적용 투자자
- Mark Minervini$VCP/SEPA를 활용한 성장주 투자자
- 미국/$韩国/$대형주 중심 거래자

**주의점:**
- 추세추종 (Trend Following) 전략에는 별도 구현 필요
- 펀더멘털 데이터 확보가 필수 - 데이터 부재시 분석 제한
-機械적 손절 ($7-8%) 규칙 준수 필요

**전체 평가:** CANSLIM/VCP 기반 성장주 투자에 최적화된精心 설계된 프로그램. 추세추종專門 기능은 부족하나基本的이며 체계적인 투자 프레임워크를 제공
