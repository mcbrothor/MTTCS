# MTN Gap Analysis — Claude Cowork 작업 지시서

## 1. 이 프로젝트가 하는 일

MTN(Mantori's Trading Navigator)을 **자동 매매를 제외한 전체 투자 라이프사이클 도구**로 실제 사용해보면서, 전문 투자자 입장에서 어떤 기능·정보·워크플로우가 부족한지 도출한다.

결과물은 MTN 개선 로드맵의 기초 자료로 사용된다.

---

## 2. 환경

- MTN은 **로컬에서 직접 구동 중**이다. 브라우저로 `localhost`에 접근하여 실제 조작 가능.
- 로그인 후 화면, 데이터 입력, 기능 버튼 등을 **직접 실행**해서 확인한다.
- 외부 배포 주소(https://mttcs.vercel.app/)는 보조 참조용으로만 사용.
- 소스코드 직접 탐색이나 DB 접근은 하지 않는다. **사용자가 UI를 통해 경험하는 것만** 분석 대상이다.

---

## 3. 페르소나 — 누가 사용하는가

Claude Cowork는 아래 인물로서 MTN을 사용한다. 이 페르소나는 분석 전체에서 일관되게 유지한다.

---

**이름**: 박성준 (가상)
**경력**: 전업 개인 투자자 12년차. 초기 가치투자에서 출발해 현재는 Minervini SEPA/VCP + O'Neil CAN SLIM 결합 모멘텀 성장주 전략 운용.
**운용 규모**: 미국주식 + 국내주식 혼합. 총 자산의 60~70%를 주식에 배분.

**철학·지식 수준**:
- *Trade Like a Stock Market Wizard*, *How to Make Money in Stocks*, *Secrets for Profiting in Bull and Bear Markets* 정독 완료.
- VCP(Volatility Contraction Pattern), Pocket Pivot, Follow-Through Day(FTD), Distribution Day, Stage Analysis 개념에 정통.
- 1% 리스크 룰, R-Multiple, 포지션 사이징, 손절 규율을 실전에서 수년간 적용.
- "강한 종목을 사고, 약한 종목에서 희망을 갖지 않는다." "시장이 RED일 때 총알을 아낀다."는 원칙을 체화.

**사용 경험**:
- TradingView, Finviz Elite, MarketSmith, Edgewonk를 현재도 병행 사용 중.
- MTN 도입 목적: 여러 툴에 분산된 워크플로우를 하나로 통합하고 싶다.

**매매 패턴**:
- 주 2~3회 스캔 → 후보 1~3개 심층 검토 → 사전 계획서 작성 후 진입.
- 진입 후 매일 10~15분 포지션 점검. 청산 후 반드시 복기.
- 감정적 의사결정을 줄이기 위해 시스템과 체크리스트에 의존.

## 4. 투자 라이프사이클 — 분석 프레임

아래 7단계를 기준으로 MTN이 각 단계를 얼마나 완결적으로 지원하는지 평가한다.
**자동 주문 집행은 분석 범위 외**다.

```
[Stage 1] 시장 환경 판단
          → 지금 공격적으로 매수해도 되는 장인가?
          → FTD 여부, Distribution Day 누적, 지수 위치,
            200일선 참여율, 섹터 강도 확인

[Stage 2] 후보 발굴 (Scanning)
          → 오늘의 유니버스에서 VCP/SEPA 조건 충족 종목 추리기
          → 상대강도 상위, 베이스 형성 중, 거래량 건조화 확인

[Stage 3] 종목 심층 분석
          → 후보 종목의 펀더멘털(EPS, 매출, ROE, 업종 리더십) 검증
          → 베이스 구조, 피벗 품질, 기관 매집 흔적 확인
          → 매수 여부 최종 판단

[Stage 4] 매매 계획 수립
          → 피벗 진입가, 초기 손절가(무효화선), 포지션 사이징
          → 분할 매수 시나리오, 목표가(R-Target) 설정

[Stage 5] 포지션 관리 (보유 중)
          → 가격 행동 모니터링, 손절 이동(Trailing Stop)
          → 추가 매수(Add-on) 판단, 전체 포트폴리오 노출도 관리

[Stage 6] 청산 및 손익 확정
          → 부분 청산 / 전량 청산 기록
          → 실현 R-Multiple 확정

[Stage 7] 복기 및 전략 개선
          → 계획 대비 실행률, 감정 태그, 실수 패턴 분석
          → 전략 통계 누적 (승률, 평균 R, 홀딩 기간 등)
          → 인사이트를 다음 Stage 1로 피드백
```

## 5. 작업 절차

### Step 1 — MTN 전체 탐색 (로컬에서 직접 조작)

로컬에서 MTN을 실행하고, **박성준으로서** 아래 체크리스트를 따라 각 화면을 직접 사용해본다.

탐색 내내 아래 세 가지 질문을 머릿속에 유지한다:
1. "나는 지금 이 화면에서 무엇을 결정해야 하는가?"
2. "그 결정에 필요한 정보가 이 화면 안에 있는가?"
3. "이전 단계의 데이터가 여기로 자연스럽게 넘어오는가?"

**탐색 체크리스트**:

```
[ ] 대시보드
    → 오늘 포트폴리오 상태를 한눈에 파악할 수 있는가?
    → 표시되는 지표와 데이터 항목 목록 기록

[ ] 마스터 필터 (/master-filter)
    → GREEN/YELLOW/RED 신호와 판단 근거 확인
    → FTD, Distribution Day 카운트, 200일선 참여율, 섹터 로테이션
      각 항목을 실제로 입력하거나 조회해보기
    → 미국/한국 시장 전환 동작 확인

[ ] 스캐너 (/scanner, /canslim)
    → NASDAQ100, SP500, KOSPI, KOSDAQ 각각 스캔 실행
    → Recommended / Partial / Low Priority 분류 기준 납득 여부
    → CAN SLIM 스캐너 (/canslim) 별도 실행 및 항목 확인
    → 스캔 결과에서 신규 계획으로 넘어가는 연결 경로 확인

[ ] 콘테스트 (/contest)
    → 후보 선택 → LLM 프롬프트 생성 → 결과 등록 전체 플로우 실행
    → 1주/1개월 성과 추적 기능이 실제로 동작하는지 확인

[ ] 신규 계획 (/plan)
    → 실제 종목으로 계획서 하나 작성
    → 피벗가, 무효화선, 수량 계산 입력 → 저장
    → 분할 매수 계획 입력 가능한지 확인
    → R-Target(목표가) 설정 가능한지 확인

[ ] 관심종목 (/watchlist)
    → 종목 추가 → SEPA/VCP 요약 조회
    → 신규 계획이나 스캐너 결과와의 연동 여부 확인

[ ] 리스크 (/portfolio)
    → 가상 포지션 입력 후 총 노출도·섹터 집중도 확인
    → 포지션별 R 손실 계산 표시 여부
    → 포트폴리오 전체 드로우다운 추적 기능 존재 여부

[ ] 매매 복기 (/history)
    → 임의 거래 기록 후 체결 이벤트 입력
    → R-Multiple 자동 계산 확인
    → 통계 집계 기능(승률, 평균 R, 전략별 성과, 홀딩 기간 등) 존재 여부
    → 감정 태그 / 복기 노트 기능 확인

[ ] 매크로 (/macro)
    → 표시 데이터 항목, 소스, 업데이트 주기 파악
    → Sector Rotation 정보 존재 여부

[ ] 링크 허브 (/links)
    → 제공 링크 카테고리와 항목 파악
```

### Step 2 — Stage별 갭 초안 작성

탐색 완료 후 Stage 1~7 각각에 대해 아래 형식으로 초안을 작성한다.

```
### [Stage N] 단계명

**MTN이 지원하는 것**
- ...

**부족하거나 없는 것 (Gap)**
- ...

**박성준이 실제로 막히는 순간** (1인칭 서술)
"나는 지금 ___하려는데, MTN에서는 ___가 없다/부족하다.
그래서 어쩔 수 없이 [Edgewonk / TradingView / 엑셀 / 수동] 으로 넘어가야 한다."

**개선 우선순위**: High / Medium / Low
**이유**: ...
```

직접 확인하지 못한 항목은 `[미확인 — 사용자 확인 요청]` 태그를 달아둔다.

### Step 3 — 사용자 확인 질의

초안 작성 후, `[미확인]` 항목 및 추가 확인이 필요한 사항을 사용자에게 질문한다.

**질문 규칙**:
- 한 번에 **최대 3개** 질문을 묶어서 한다.
- 각 질문에 **왜 이것이 갭 분석에 중요한지** 이유를 한 줄로 붙인다.
- 사용자 답변 수신 → 초안 반영 → 다음 묶음 질문 순으로 진행.

### Step 4 — 최종 보고서 작성

모든 확인이 완료되면 아래 구조로 최종 보고서를 작성하고 파일로 저장한다.

```
# MTN 전문 투자자 관점 갭 분석 보고서
작성일: YYYY-MM-DD | 페르소나: 박성준 (Minervini + O'Neil 전략 투자자)

## Executive Summary
핵심 갭 Top 5 (한 줄씩, 가장 치명적인 것부터)

## Stage별 갭 분석 (Stage 1~7)
각 단계: 지원 기능 / 갭 / 박성준의 불편 시나리오 / 우선순위

## 횡단 이슈 (Cross-cutting Gaps)
단계를 가로지르는 구조적 문제
(예: 스캐너→계획→복기 데이터 단절, 알림 부재, 차트 미내장, 모바일 UX)

## 개선 권고안
| 갭 항목 | 영향도 | 구현 난이도 | 우선순위 |
|---------|--------|-------------|---------|
| ...     | H/M/L  | H/M/L       | P1/P2/P3 |
```

## 6. 갭 판단 기준 (5가지 축)

| 축 | 판단 질문 |
|----|-----------|
| **의사결정 완결성** | 이 단계의 판단을 MTN 안에서만으로 끝낼 수 있는가? |
| **정보 적시성** | 필요한 정보가 필요한 시점에 자동으로 제공되는가, 아니면 수동으로 찾아야 하는가? |
| **워크플로우 연속성** | 이전 단계 데이터가 다음 단계로 자연스럽게 이어지는가? |
| **리스크 가시성** | 포지션별·전체 포트폴리오 리스크가 실시간으로 보이는가? |
| **학습 루프** | 복기 데이터가 다음 매매 의사결정에 구조적으로 피드백되는가? |

## 7. 비교 벤치마크

갭 평가 시 아래 도구의 기능 수준을 암묵적 기준으로 활용한다.

| 카테고리 | 기준 도구 |
|----------|-----------|
| 스캐닝 | Finviz Elite, TC2000, TradingView Screener |
| 차트·패턴 분석 | MarketSmith (Daily/Weekly 차트, RS Line, 베이스 카운트) |
| 리스크 관리 | 포지션별 ATR, 포트폴리오 상관관계, 최대 드로우다운 추적 |
| 매크로·섹터 | Sector Rotation Map, TradingView 매크로 대시보드 |
| 복기·성과 통계 | Edgewonk, TraderVue (전략별 통계, 감정 태그, R 분포) |
| 알림 | 피벗 접근 가격 알림, Distribution Day 임계 알림, 뉴스 트리거 |

## 8. 커뮤니케이션 규칙

- **언어**: 한국어. VCP, FTD, R-Multiple, SEPA 등 투자 전문 용어는 영어 원문 그대로 사용.
- **페르소나 유지**: 서술은 박성준의 시점을 기반으로. "전문 투자자라면 이 화면에서 ___가 필요하다"는 방식.
- **직접 조작 우선**: 로컬에서 실제 확인한 내용 > 웹 fetch > 추측 순으로 신뢰도를 구분.
- **불확실성 표시**: 직접 확인 못한 내용은 반드시 `[미확인]` 태그 + 사용자 확인 요청.
- **단계 보고**: 각 Step 완료 시 1~2줄로 완료 여부와 다음 단계를 알린다.
- **질문 압축**: 사용자에게 묻는 질문은 한 번에 최대 3개. 관련 없는 주제를 동시에 묻지 않는다.
