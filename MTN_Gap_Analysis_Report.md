# MTN 전문 투자자 관점 갭 분석 보고서

작성일: 2026-04-21 | 페르소나: 박성준 (Minervini SEPA/VCP + O'Neil CAN SLIM 전략 투자자)

> **분석 방법**: 소스코드(Next.js 컴포넌트, API 라우트, 타입 정의) 직접 탐독을 통해 UI 기능 및 데이터 흐름을 파악함. 로컬 브라우저 직접 접근은 환경 제약으로 불가, 코드 기반 분석으로 대체. 초기 `[미확인]` 항목은 이후 관련 API 라우트 및 라이브러리 코드 재분석을 통해 모두 확인 완료.
>
> **2차 검증 완료일**: 2026-04-21 — `app/api/master-filter/route.ts`, `app/api/telegram-webhook/route.ts`, `app/api/trade-stop-events/route.ts`, `app/api/trade-exit-rules/route.ts`, `app/api/trades/route.ts`, `lib/finance/core/trade-metrics.ts` 직접 분석.

## Executive Summary — 핵심 갭 Top 5

1. **차트 미내장**: MTN 전 화면에 걸쳐 인터랙티브 주가 차트(일봉/주봉, RS Line, 베이스 카운트)가 전혀 없다. VCP 패턴 판정이 텍스트 수치만으로 제시되어 눈으로 확인이 불가하다.

2. **Stage 3→4 데이터 단절**: 스캐너에서 발굴한 종목을 Plan 화면으로 이동할 때 RS 점수, CAN SLIM 필라 결과, 피벗 품질 판정 등 스캐너 데이터가 계획서에 자동 연결되지 않는다. 중간 정보가 소실된다.

3. **포지션 실시간 모니터링 부재 (Stage 5)**: 보유 중 포지션에 대한 현재가 자동 갱신, Trailing Stop 상태 표시, 현재 R 실시간 업데이트, 알림 기능이 없다. 매일 10~15분 점검 루틴을 MTN 안에서 수행할 수 없다.

4. **복기 통계 미집계 (Stage 7)**: 전략별(VCP, CAN SLIM) 승률·평균 R·홀딩 기간 분포, 실수 태그별 빈도 분석, 계획 대비 실행률 추세가 집계 화면으로 제공되지 않는다. 학습 루프가 닫히지 않는다.

5. **알림 시스템 전무**: 피벗 접근 가격 알림, Distribution Day 임계치(5일) 도달 알림, 손절선 근접 알림 등 어떤 능동적 알림도 없다. 박성준은 스스로 매일 수동으로 확인해야 한다.

## Stage별 갭 분석

---

### [Stage 1] 시장 환경 판단

**MTN이 지원하는 것**
- 마스터 필터(`/master-filter`): GREEN/YELLOW/RED 3단계 시장 판정, P3 신뢰 점수(0~100점) 자동 계산
- 판단 근거 7개 지표: 추세(50/200일선), 내부강도(신고가-신저가), 변동성(VIX), 유동성, FTD(Follow-Through Day), Distribution Day 카운트, 섹터 로테이션
- 전체 섹터 ETF 20일 수익률 순위표 및 Risk-on/Defensive 분류
- 매크로 대시보드(`/macro`): VIX, 달러, 채권(SHY/TLT/HYG/IEF), 주요 지수(QQQ/SPY/DIA/IWM/RSP), 원자재, BTC 현황 + Risk-ON/OFF 종합 판정
- 미국/한국 시장 전환 기능

**부족하거나 없는 것 (Gap)**
- **FTD/Distribution Day 자동 계산 — 단, 기준값 차이 있음**: `app/api/master-filter/route.ts` 확인 결과 FTD와 Distribution Day는 Yahoo Finance EOD 데이터를 기반으로 완전 자동 계산됨. FTD 기준: 최근 30세션 내 4% 이상 하락 구간 탐지 후, 4일차+ 중 **+1.25% 이상 상승 + 거래량 > 전일** 조건 충족일로 판정. 단, O'Neil 원전 기준(+1.7% 이상, 4~7일차)과 임계값이 다르며 사용자 조정 불가
- **Distribution Day 세부 내역 표시 없음**: 누적 카운트(숫자)는 표시되지만, 어느 날짜에 몇 %로 발생했는지 날짜별 내역 리스트가 없음
- **RS Line(상대강도선) 차트 미제공**: MarketSmith의 RS Line(종목 주가 vs S&P 500 상대강도 추이선)이 없음. RS 숫자 점수만 제공
- **지수 일봉/주봉 차트 미내장**: S&P 500, NASDAQ 지수 자체의 가격 차트를 MTN 내에서 볼 수 없음
- **섹터별 개별 종목 드릴다운 없음**: 섹터 로테이션 테이블에서 특정 섹터를 클릭해도 해당 섹터 강세 종목 목록으로 연결되지 않음
- **한국 시장 FTD/Distribution Day 알고리즘 동일**: `^KS200` (KOSPI 200 ETF) 데이터에 미국과 동일한 1.25% 기준 알고리즘을 적용하는 것으로 코드에서 확인됨. 한국 시장 고유 기준(예: KOSPI 특성상 거래량 패턴 차이)은 반영되지 않음

**박성준이 실제로 막히는 순간**
> "나는 오늘 Distribution Day가 4일째인지 5일째인지 정확히 확인하고 싶은데, MTN 화면에는 숫자만 나오고 어느 날 발생했는지 내역이 없다. FTD가 진짜로 확인된 건지도 야후파이낸스 지연 데이터 기반이라 믿기 애매해서 결국 MarketSmith를 켜야 한다."

**개선 우선순위**: High  
**이유**: Stage 1은 모든 이후 단계의 전제조건. 신호의 신뢰도가 낮으면 전체 투자 판단의 품질이 하락한다.

### [Stage 2] 후보 발굴 (Scanning)

**MTN이 지원하는 것**
- VCP 마스터 스캐너(`/scanner`): NASDAQ100, SP500, KOSPI200, KOSDAQ150 유니버스 대상 SEPA/VCP 조건 자동 평가
- Recommended / Partial / Low Priority 3단계 분류 및 이유 표시
- RS Rating 점수, 피벗가, 피벗까지 거리(%), VCP 점수, 베이스 패턴 타입 표시
- CAN SLIM 스캐너(`/canslim`): 7 Pillar(M·C·A·N·S·L·I) 평가, TIER 1/WATCHLIST/SHORT_TERM 분류, 신뢰도(HIGH/MEDIUM/LOW) 표시
- 스캔 결과에서 콘테스트로 후보 직접 이동(플로팅 툴바)
- 스캔 결과에서 관심종목 추가 가능
- 스캔 결과 로컬 스냅샷 저장(localStorage)으로 재실행 없이 조회 가능

**부족하거나 없는 것 (Gap)**
- **차트 없음**: 스캔 결과 종목의 일봉/주봉 차트를 MTN 내에서 볼 수 없음. VCP 패턴 판정이 텍스트·숫자만으로 표시되어 육안 확인 불가. VcpDrilldownModal이 존재하나 실제 차트는 미포함
- **RS Line 신고가 여부 표시 미흡**: `rs_line_new_high` 필드가 데이터 모델에 존재하나 스캐너 테이블에 컬럼으로 표시되지 않음
- **거래량 건조화 시각화 없음**: volumeDryUpScore는 계산되나 최근 N일 거래량 추이 차트가 없어 "VCP 수축 국면에서의 거래량 패턴"을 눈으로 보고 판단할 수 없음
- **스캔 결과 자동 갱신 없음**: 수동으로 스캔 버튼을 눌러야 하며, 특정 종목이 피벗 근처에 접근했을 때 알림 없음
- **필터 커스터마이징 불가**: RS 점수 범위, VCP 점수 임계값, 거리(%) 상한 등을 사용자가 직접 설정하는 필터 UI 없음
- **Pocket Pivot 개별 표시 없음**: Pocket Pivot 점수는 계산되나 결과 테이블에 컬럼이 없음
- **스캐너 → 계획서 직접 연결 없음**: 스캔 결과에서 "신규 계획 작성" 버튼이 없음. 관심종목 → 계획서 경로는 있으나, 스캐너 데이터(RS, VCP score, 피벗가)가 계획서에 자동 채워지지 않음

**박성준이 실제로 막히는 순간**
> "스캐너에서 NVDA가 VCP Recommended로 떴는데, 정말로 피벗 근처에서 거래량이 말라붙었는지 주봉 차트를 보고 싶다. MTN에는 차트가 없으니 TradingView를 열어야 한다. 두 화면을 오가다 보면 분석의 흐름이 끊긴다."

**개선 우선순위**: High  
**이유**: 차트 없는 스캐너는 "후보 목록 제공"에 그칠 뿐 "후보 검증"을 MTN 안에서 완결하지 못한다.

### [Stage 3] 종목 심층 분석

**MTN이 지원하는 것**
- 신규 계획 화면(`/plan`): 티커 입력 시 SEPA 판정(통과/실패 근거 요약), VCP 분석(Grade, Score, Detail 텍스트), 리스크 계산 자동 제공
- SEPA Evidence: 각 항목(추세, RS, EPS 가속, 베이스 등)의 통과/실패 상태와 이유 텍스트 표시
- VCP Analysis Panel: 수축 점수, Pocket Pivot 점수, 돌파 거래량 상태, 세부 내용 텍스트
- 관심종목(`/watchlist`): 클릭 시 최근 SEPA/VCP 요약(Grade, Score, 거래량 신호 등급) 즉시 조회

**부족하거나 없는 것 (Gap)**
- **펀더멘털 데이터 직접 표시 없음**: EPS 증가율 3~5분기 추이, 매출 성장률, ROE, 부채비율, 기관 보유 변화 수치 등이 화면에 직접 표시되지 않음. SEPA 판정 결과(통과/실패)로만 보여줌
- **베이스 구조 시각화 없음**: 베이스 카운트(1st stage, 2nd stage), 피벗 품질, 핸들 형성 여부를 차트로 확인 불가
- **기관 매집 흔적 확인 불가**: O'Neil의 'I' (Institutional Sponsorship) 항목은 CAN SLIM 필라에 포함되어 있으나, 실제 기관 보유 비율 변화 데이터 소스가 Yahoo Finance 지연 데이터에서 파생된 근사값으로 추정되며 정밀한 13-F 기반 기관 보유 변화 추적은 불가
- **뉴스·실적 발표 일정 없음**: 다음 어닝 발표일, 최근 주요 뉴스가 MTN 내에서 제공되지 않음
- **업종 내 상대 위치 없음**: 스캐너 RS 랭킹은 해당 유니버스(NASDAQ100, SP500 등) 전체 기준 상대값이며, 동일 업종 내 리더십 순위는 별도 제공되지 않음
- **Short Interest / 공매도 비율 없음**: 주요 성장주 투자에서 고려하는 공매도 비율 데이터 부재

**박성준이 실제로 막히는 순간**
> "MTN에서 SEPA PASS가 떴는데 EPS가 실제로 몇 분기 연속 가속 성장했는지, 매출 증가율이 어떤지 숫자를 직접 보고 싶다. '통과'라는 판정 결과만 보여줘서는 직접 판단을 내릴 수 없다. 결국 Finviz Elite나 회사 IR 페이지를 열게 된다."

**개선 우선순위**: High  
**이유**: 전문 투자자는 판정 결과를 맹목적으로 신뢰하지 않는다. 근거 데이터를 직접 보고 최종 판단하는 단계가 Stage 3의 핵심이다.

### [Stage 4] 매매 계획 수립

**MTN이 지원하는 것**
- 리스크 계산기: 총 자본, 허용 손실 비율(%) 입력 → 최대 허용 손실 금액, ATR 참고값, 피벗 진입가, 초기 손절가, 총 수량 자동 산출
- 분할 매수 3단계(E1/E2/E3) 계획: 각 단계별 기준가, 수량, 트레일링 스탑 기준 표시
- Trailing Stop 3단계 자동 계산(initial / afterEntry2 / afterEntry3)
- Centaur 체크리스트: 6개 항목(SEPA, 리스크, 진입, 손절, 청산, 심리) 체크 후 저장 가능
- 계획 저장 시 SEPA 근거, VCP 분석, 리스크 수치 함께 DB 보관

**부족하거나 없는 것 (Gap)**
- **R-Target(목표가) 설정 필드 없음**: 1R, 2R, 3R 목표가를 계획서에 명시하는 기능이 UI에 없음. 단, `app/api/trade-exit-rules/route.ts` 분석 결과 `trade_exit_rules` 테이블과 완전한 CRUD API가 구현되어 있고 `R_MULTIPLE`, `GAIN_PCT`, `PRICE`, `MANUAL` 4가지 트리거 타입과 `exit_fraction`(부분 청산 비율)까지 지원함. **API 인프라는 완성되어 있으나 UI 컴포넌트가 존재하지 않는 상태** — R-Target 기능이 개발 중도에 멈춰있음
- **시나리오별 계획 작성 불가**: "피벗 돌파 즉시 진입" vs "되돌림 후 재진입" vs "2차 진입 조건" 등 복수 시나리오를 하나의 계획서에 담을 수 없음
- **계획서 차트 뷰 없음**: 계획서를 작성하면서 진입가/손절가/목표가를 가격 차트 위에 시각적으로 표시하는 기능 없음
- **계획 수정 이력 없음**: 저장 후 계획을 수정하면 이전 버전이 사라짐. 계획 변경 사유를 남기는 필드 없음
- **Short 포지션 미지원**: `app/api/trades/route.ts` 확인 결과 `direction` 필드는 `body.direction || 'LONG'` 으로 기본값이 LONG이며, API는 다른 값도 수용하나 계획서 UI(`/plan`)에 direction 입력 필드가 존재하지 않아 사실상 LONG 전용. 공매도 전략 계획서 작성 불가
- **ATR 배수 기반 손절가 직접 입력 없음**: 피벗가와 손절가를 각각 입력하면 리스크 금액이 계산되는 방식이며, "ATR × 배수 = 손절가" 방식의 직접 설정 UI는 없음. ATR 참고값은 표시되나 이를 손절가에 직접 연동하는 기능 없음

**박성준이 실제로 막히는 순간**
> "계획서를 저장하면서 '1R에서 절반 청산, 2R에서 나머지 청산'이라는 목표가 시나리오를 함께 기록하고 싶은데 MTN에는 목표가 입력 필드 자체가 없다. R-Target 없이는 진입-청산 전체 시나리오가 반쪽짜리다. 어쩔 수 없이 엑셀에 따로 정리한다."

**개선 우선순위**: High  
**이유**: R-Target 없이는 청산 판단 기준이 주관적이 되고, 복기 시 R-Multiple 계산도 후행적으로만 가능하다.

### [Stage 5] 포지션 관리 (보유 중)

**MTN이 지원하는 것**
- 포트폴리오 리스크 화면(`/portfolio`): 총 자산, 투자금, 현금, 오픈 리스크, 포지션 수 표시
- 섹터별 노출도(금액 및 %) 바 차트
- 거래 체결 기록 패널: E1/E2/E3 단계별 매수·매도 체결 수동 입력, 평균 진입가·미실현 손익·미실현 R 계산
- 미실현 R 및 현재 R 실시간 표시 (단, 현재가 수동 입력 필요)

**부족하거나 없는 것 (Gap)**
- **현재가 자동 조회됨 — 단, Pull 방식 한계 있음**: `app/api/trades/route.ts` 확인 결과 GET 요청 시마다 ACTIVE 포지션의 현재가를 자동 조회함. 미국 종목은 Yahoo Finance 배치 쿼리, 한국 종목은 KIS API 개별 호출. 조회된 현재가는 `trade-metrics.ts`를 통해 unrealizedPnL과 unrealizedR 계산에 즉시 반영됨. **단, 이는 페이지 로드/새로고침 시점의 Pull 방식**이며 장중 자동 Push(WebSocket/SSE) 갱신이나 백그라운드 폴링은 없음. 사용자가 페이지를 열지 않으면 업데이트되지 않음
- **Trailing Stop 현황 대시보드 없음**: 계획서에서 설정한 E1/E2/E3 단계별 트레일링 스탑이 현재가 대비 어느 위치에 있는지 한눈에 보이는 화면 없음. `app/api/trade-stop-events/route.ts` 확인 결과 손절 변경 이력 API(`trade_stop_events` 테이블, INITIAL/TEN_WEEK_MA/HIGH_WATERMARK/MANUAL/PYRAMID 소스 분류)가 완전히 구현되어 있으나 이를 표시하는 UI 컴포넌트가 없음 — R-Target과 마찬가지로 **API 완성, UI 미완성** 상태
- **Add-on(추가 매수) 조건 판단 지원 없음**: Pocket Pivot 발생 여부, 2번째 베이스 형성 여부 등 추가 매수 시그널을 자동으로 알려주는 기능 없음
- **포트폴리오 전체 드로우다운 추적 없음**: 최고점 대비 현재 포트폴리오 드로우다운 %가 계산되어 표시되지 않음
- **종목 간 상관관계 표시 없음**: 보유 중인 복수 종목이 동일 섹터/테마에 집중되어 있는지 위험 집중도를 정량적으로 보여주지 않음
- **알림 없음**: 손절선 근접(예: 손절가의 3% 이내), 일중 -8% 발생, Distribution Day 5일 도달 등 어떤 조건 알림도 없음

**박성준이 실제로 막히는 순간**
> "아침에 MTN을 열어서 오늘 NVDA가 초기 손절가에 얼마나 근접했는지, E2 추가 매수 조건인 Pocket Pivot이 왔는지 확인하려는데 MTN은 이걸 실시간으로 보여주지 않는다. 결국 TradingView 알림을 세팅하고 그걸 보고 MTN에 수동으로 기록하는 이중 작업을 하고 있다."

**개선 우선순위**: High  
**이유**: Stage 5는 투자 수익의 대부분이 결정되는 구간이다. 적시 정보 없이는 감정적 의사결정이 개입될 가능성이 높다.

### [Stage 6] 청산 및 손익 확정

**MTN이 지원하는 것**
- 체결 패널에서 EXIT 기록(매도 수량, 가격, 수수료) 입력 가능
- 부분 청산 / 전량 청산 모두 기록 가능
- 청산 후 실현 손익(금액) 자동 계산
- R-Multiple 자동 계산: `(실현 손익) / (계획된 최대 리스크)`로 산출

**부족하거나 없는 것 (Gap)**
- **청산 사유 태그 필드 없음**: "손절", "목표가 도달", "시장 RED 전환", "기술적 이탈" 등 청산 이유를 드롭다운/태그로 기록하는 필드 없음. 체결 기록(`TradeExecution`)에는 자유형식 `note` 텍스트 필드만 있어, 나중에 청산 유형별 집계나 필터링이 불가
- **부분 청산 R 계산 세분화 부족**: E1 물량만 청산한 경우의 R vs E3까지 쌓은 전체 포지션 기준 R을 구분 계산하는 UI 없음
- **청산가 자동 조회 없음**: 청산 당일 종가를 자동으로 가져오는 기능 없음. 사용자가 직접 가격을 찾아 입력해야 함
- **세금/슬리피지 고려 순 R 없음**: 수수료는 입력 가능하나 세금(양도소득세, 증권거래세)을 고려한 세후 실현 R 계산 없음

**박성준이 실제로 막히는 순간**
> "오늘 청산 후 R-Multiple을 기록하고 싶은데 왜 청산했는지 사유를 남기는 드롭다운이 없다. 나중에 복기할 때 '이 매매가 계획된 손절인지, 감정적 조기 매도인지' 구분이 안 된다."

**개선 우선순위**: Medium  
**이유**: 현재도 핵심 기능(체결 기록, R 계산)은 작동한다. 다만 청산 사유 기록 없이는 Stage 7 학습 루프가 약해진다.

### [Stage 7] 복기 및 전략 개선

**MTN이 지원하는 것**
- 복기 화면(`/history`): 저장된 전체 거래 목록 표시
- 거래별 복기 패널: 셋업 태그(VCP/SEPA/돌파/실적/추세/관심종목), 실수 태그(추격매수/손절지연/비중초과/조기매도/계획미준수/진입지연)
- 규율 점수(0~100점) 수동 입력, 사후 복기 메모, 다음 개선 액션 필드
- 대시보드에서 승률, 평균 R, 기대값(Expectancy R), 계획 준수율, SEPA 통과율, Equity Curve 차트 표시

**부족하거나 없는 것 (Gap)**
- **전략별 통계 분류 없음**: VCP 셋업 vs CAN SLIM 셋업 vs 기타 셋업별 승률·평균 R·홀딩 기간을 분리 집계하는 화면 없음
- **실수 태그 분석 차트 없음**: '손절지연' 태그가 붙은 거래가 몇 건이고 평균 R이 얼마인지 등 태그별 집계가 없음
- **홀딩 기간 분포 없음**: 평균 홀딩 기간, 단기(~5일) / 중기(5~20일) / 장기(20일+) 분포가 없음
- **계획 대비 실행률 추세 없음**: Plan Adherence가 매매별로 있지만, 월별·분기별 추세 차트가 없음
- **최대 연속 손실(Max Drawdown 계열) 통계 없음**: 연속 손실 횟수, 최악의 드로우다운 구간 등 위험 관리 관련 고급 통계 없음
- **콘테스트 복기와 실제 매매 복기의 연결 없음**: 콘테스트에서 분석한 종목을 실제로 거래했을 때의 결과가 콘테스트 히스토리에 자동 연결되지 않음
- **인사이트 → Stage 1 피드백 구조 없음**: 복기에서 도출된 인사이트(예: "RED 시장에서 진입한 거래 승률 20%")가 마스터 필터나 스캐너 설정에 자동 반영되는 구조 없음

**박성준이 실제로 막히는 순간**
> "이번 달에 VCP 셋업으로 5번 거래해서 승률이 어땠는지, 실수 태그 중 '손절지연'이 몇 번이나 나왔는지 MTN이 보여주지 않는다. Edgewonk에서는 이걸 자동으로 분석해줬는데, MTN에서는 개별 거래 기록을 내가 직접 엑셀로 내보내서 계산해야 한다. 학습 루프가 구조적으로 닫히지 않는다."

**개선 우선순위**: High  
**이유**: 학습 루프 없는 복기는 단순 일지에 불과하다. 통계 피드백이 없으면 전략 개선이 주관에 의존하게 된다.

## 횡단 이슈 (Cross-cutting Gaps)

### 1. 차트 전면 부재 (가장 심각한 구조적 결함)
MTN의 전 화면(스캐너, 계획서, 관심종목, 복기)에 걸쳐 인터랙티브 주가 차트가 단 하나도 없다. VCP/SEPA 판정이 텍스트와 숫자로만 제공된다. 모멘텀 성장주 전략의 핵심인 "패턴을 눈으로 보고 판단"이 MTN 안에서는 불가능하다. MarketSmith, TradingView 의존도가 필연적이다.

### 2. 스캐너 → 계획 → 복기 데이터 단절
스캐너에서 발굴한 종목의 RS 점수, CAN SLIM 필라 결과, VCP 점수, 피벗가 데이터가 계획서에 자동 이전되지 않는다. 계획서에서 확정한 진입가/손절가/수량이 포지션 관리 화면에 실시간으로 연동되지 않는다. 각 단계가 데이터로 연결되지 않고 독립적으로 존재한다.

### 3. 알림(Alert) 시스템 전무 — Telegram 봇은 구현됐으나 알림 용도 아님
피벗 접근 알림, Distribution Day 임계치 알림, 손절선 근접 알림, 마스터 필터 상태 변화 알림 등 어떤 종류의 능동적 푸시 알림도 없다. `app/api/telegram-webhook/route.ts` 직접 확인 결과 Telegram 봇은 `grammy` 라이브러리 기반으로 완전히 구현되어 있으나, 지원 명령어는 `/help`, `/status`(ACTIVE/PLANNED 거래 조회), `/close`(거래 청산 처리), `/cancel`(거래 취소) 4가지뿐이다. **이는 거래 관리 봇이지 가격 알림 봇이 아니다**. 조건 충족 시 사용자에게 메시지를 보내는 능동적 알림(Push) 기능은 구현되지 않았다.

### 4. 모바일 UX 미최적화
투자자는 장중에 스마트폰으로 포지션을 점검한다. 현재 MTN은 Tailwind 반응형 레이아웃(`md:grid-cols-*`, `min-w-[720px]` 테이블 등)을 갖추고 있으나, 체결 기록 테이블이 `min-w-[720px]` 고정폭으로 정의되어 있어 모바일 화면에서 가로 스크롤이 발생하는 구조임. 별도 모바일 앱은 존재하지 않으며, Telegram 봇(`/status` 명령)이 사실상 유일한 모바일 접근 수단이나 조회 기능만 제공함.

### 5. 외부 도구 의존도 해소 실패
MTN의 도입 목적이 "여러 툴에 분산된 워크플로우 통합"이지만, 차트(TradingView), 펀더멘털(Finviz/MarketSmith), 통계 복기(Edgewonk), 알림(TradingView 알림)에 대한 의존이 현재도 필수적으로 남아있다.

### 6. 데이터 소스 불안정성 및 지연
Yahoo Finance 지연 데이터 기반으로 운영되며, 일부 항목은 fallback 처리가 있음. 실시간 데이터 미제공으로 인해 장중 의사결정 지원이 근본적으로 제한된다.
