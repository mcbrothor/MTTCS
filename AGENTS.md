# AGENTS.md

## 에이전트 행동 원칙

의도가 명확하면 즉시 실행한다. 불필요한 확인 질문을 하지 않는다.

**반드시 확인 후 진행:**
- 파괴적 작업 (삭제, 덮어쓰기, drop table)
- 범위가 불명확한 대규모 변경
- 결과가 되돌릴 수 없는 아키텍처 결정

**묻지 않고 판단:**
- 코드 스타일 → 기존 컨벤션 따름
- 네이밍 → 컨텍스트에서 추론
- 구현 방식 → 명확히 나은 방향 선택

---

## 계획 (Planning)

다단계 태스크 시작 전, 아래 형식으로 계획을 출력한다.
중단 신호가 없으면 바로 실행한다.

```
PLAN: [태스크명]
├── Wave 1 (병렬): task_a | task_b | task_c
├── Wave 2 (병렬): task_d | task_e
└── Wave 3 (순차): task_f → task_g
Checkpoint: [다음 Wave 진입 전 검증 항목]
```

분해 기준은 파일·모듈이 아니라 **의존성**이다.
공유 상태가 없는 태스크는 같은 Wave에 넣는다.

---

## 실행 (Execution)

### 병렬화
의존성이 없으면 병렬이 기본값이다.

- 여러 파일 → 동시에 읽기
- 독립적인 모듈 → 동시에 작성
- 리서치(문서/웹)와 코딩 → 항상 병렬 가능

### Checkpoint
각 Wave 완료 후 다음으로 넘어가기 전에 검증한다.

- 컴파일/lint 통과
- 테스트 실행 (가정하지 말 것, 직접 실행)
- 외부 연결 성공 (실제 endpoint hit)
- 출력 shape이 다음 Wave의 기대값과 일치

실패 시: 수정 → 재검증 → 진행.
동일한 수정이 3회 실패 → 즉시 멈추고 root cause 분석 후 접근 방식 변경.

### 파일 읽기
- 읽기 전에 목적을 먼저 명시한다
- 300줄 초과 파일 → 필요한 섹션만 읽는다
- 이미 컨텍스트에 있는 파일은 다시 읽지 않는다

## 컨텍스트 관리

컨텍스트는 유한한 자원이다. 적극적으로 압축한다.

**Wave 완료 시, 완료된 내용을 snapshot으로 교체:**
```
[완료] wave_1: auth.py, db_schema.sql, config.py
[완료] wave_2: api_routes.py, service_layer.py
[진행] wave_3: routes ↔ service 연결 중
[대기] wave_4: e2e 테스트
[이슈] 없음
```

**컨텍스트에서 제거:**
- 디스크에 저장된 파일의 전체 내용
- 해결된 에러 trace (수정 요약만 유지)
- 대체된 중간 시도들

**컨텍스트에 유지:**
- 현재 Wave 상태
- 미해결 이슈와 blocker
- Interface contract (함수 시그니처, API response shape, 이벤트 스키마)

## 출력 형식

Signal-to-noise를 최대화한다.

| 상황 | 형식 |
|------|------|
| 태스크 완료 | `✓ 파일명 — 한 줄 설명` |
| Wave 완료 | snapshot으로 압축 (위 참조) |
| 에러 | `✗ 실패 내용 → root cause → 적용한 수정` |
| 블로킹 | `? 구체적 질문 — [선택지 A / 선택지 B]` |
| 코드 변경 | 변경된 블록만 출력, 전체 파일 재출력 금지 |

**금지:**
- 실행 전 태스크 재서술
- 자명한 코드에 설명 추가
- 변경되지 않은 코드를 수정본에 포함해 재출력
- 핵심 frame 이상의 stack trace 출력

## 공통 컨벤션

기존 프로젝트 컨벤션을 추론해서 따른다. 없으면:

- **시크릿**: env var만 사용, 코드에 하드코딩 금지
- **설정값**: 로직에 magic number 삽입 금지, 외부화
- **에러 처리**: 명시적으로, `except: pass` 또는 `.catch()` 묵살 금지
- **부수효과**: 파괴적·외부 작업은 `DRY_RUN=true` 지원
- **커밋**: Wave 완료 단위로 1커밋, 명령형 메시지 ("Add auth module")
